## Quick Start

### 1. Install the desktop hub

1. Download the correct macOS `.dmg` for your machine.
2. Drag **Music Hub** into `Applications`.
3. Launch the app and allow it in **Privacy & Security** if macOS warns you.

### 2. Add your library

1. Click **Add Music Folder** on first launch.
2. Pick one or more folders that contain your music.
3. Wait for the initial scan to finish.
4. Confirm the hub is healthy in the app or at `http://localhost:38472/api/info`.

The packaged desktop hub serves on port `38472` by default. The source-development flow also uses `38472`, so dev and packaged QR/connect flows point at the same backend.

### 3. Join the mobile beta

The iPhone app is currently distributed through TestFlight, not the App Store.

1. Submit the [TestFlight sign-up form](https://docs.google.com/forms/d/e/1FAIpQLSf6CVxy2yU3rPQiVHsUYMA0pGiuM4p1VEq1fV6alAFRd4o04A/viewform?usp=dialog).
2. Install the app from the TestFlight invite email once your address is added.
3. Open the desktop hub and show the **Connect** QR.
4. Open the mobile app and scan the QR code.

The safest default is still Wi-Fi-first pairing: pair once on the same Wi-Fi, then let the phone fall back to Tailscale or the public remote later if Wi-Fi drops.

If you are pairing through a public Cloudflare URL or Tailscale route for the first time, arm pairing first in the desktop app. The desktop pairing screen now shows:

- a QR code
- a mobile deep link
- a countdown / expiry time
- a relaunch action for a fresh pairing window

If local discovery fails, enter `YOUR_IP:38472` manually in the mobile app.

If you need Windows, Linux, or a local dev build:

```bash
cd music-streaming-hub/desktop
npm install
npm run dev:all
```

## Common Setup Tasks

### Add more music folders

1. Open **Settings** in the desktop app.
2. Click **Add Music Folder**.
3. Choose another directory.
4. Music Hub will rescan automatically.

### Import Rekordbox playlists

1. In Rekordbox, export your collection as XML.
2. In Music Hub, open **Settings**.
3. Use the Rekordbox import flow and select the XML file.

### Enable remote access

- **Cloudflare Tunnel:** fastest setup, temporary public URL.
- **Tailscale:** persistent private access between your devices.

Both options live under **Settings → Remote Access** in the desktop app. Remote API reads require a bearer token after pairing, and public share pages use signed bootstrap/media sessions.

Important scope rule:

- `Shared pages only` limits the public web/share surface
- it does **not** remove private access from a phone that is already paired

## Quick Troubleshooting

### Mobile cannot connect

- Make sure the desktop app is running and the library DB is healthy.
- Re-scan the latest QR instead of reusing an old remote link.
- If the desktop returns `pairing_not_armed`, arm pairing again and reconnect.
- If the phone was already paired and Wi-Fi drops, it should keep working through the remote fallback without a new pairing.
- If manual local connect still fails, confirm the desktop is reachable on `YOUR_IP:38472`.

### Music is missing

- Confirm the folder actually contains supported audio files.
- Wait for the first scan to finish on large libraries.
- Re-open **Settings** and confirm the folder is still listed.

### The desktop app does not start

- Check whether port `38472` is already in use.
- Re-open the app from `Applications`.
- Review the logs and then follow the [Troubleshooting Guide](/troubleshooting).
